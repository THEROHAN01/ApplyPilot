# ApplyPilot — API Reference (Phase 1)

Every endpoint that exists today. Generated by reading the actual router and schema
modules under `backend/routers/` and `backend/schemas/` — not from the spec. Interactive
docs: `http://localhost:8000/docs` (Swagger UI) when the backend is running.

**Conventions**
- Base URL: `http://localhost:8000`
- Auth: a JWT **access** token in `Authorization: Bearer <token>`. Obtain via `/auth/signup` or `/auth/login`.
- All UUIDs are strings in JSON. All timestamps are ISO-8601 with timezone.
- `422 Unprocessable Entity` is returned by FastAPI/Pydantic for any body/query that fails validation (shape: `{"detail": [{"loc": [...], "msg": "...", "type": "..."}]}`).
- `429 Too Many Requests` may be returned by the rate limiter on any non-`/health` route (`> RATE_LIMIT_PER_MIN` req/60s per identity), with a `Retry-After: 60` header and body `{"detail": "Rate limit exceeded"}`.
- **Plan required:** none of the Phase 1 endpoints enforce a plan tier (plan_guard is Phase 5). The column is shown as `none` throughout.

## Health

### `GET /health`
- **Auth:** no · **Plan:** none
- **Response 200:** `{"status": "ok"}`
- **Notes:** Liveness only — does not check DB/Redis/MinIO. Exempt from rate limiting.

```bash
curl http://localhost:8000/health
# {"status":"ok"}
```

## Auth  (`/auth`)

### `POST /auth/signup`
- **Auth:** no · **Plan:** none
- **Body** (`SignupRequest`):

| Field | Type | Required | Notes |
|-------|------|----------|-------|
| `email` | string (email) | yes | Must be a valid email; unique |
| `password` | string | yes | 8–128 chars |
| `name` | string \| null | no | Display name |

- **Response 201** (`TokenPair`): `{"access_token": "...", "refresh_token": "...", "token_type": "bearer"}`
- **409 Conflict:** `{"detail": "Email already registered"}`
- **422:** validation error (bad email, password too short).

```bash
curl -X POST http://localhost:8000/auth/signup \
  -H 'Content-Type: application/json' \
  -d '{"email":"dev@applypilot.local","password":"DevTest1234!","name":"Dev"}'
# {"access_token":"eyJ...","refresh_token":"eyJ...","token_type":"bearer"}
```

### `POST /auth/login`
- **Auth:** no · **Plan:** none
- **Body** (`LoginRequest`): `{ "email": string, "password": string }`
- **Response 200** (`TokenPair`): same shape as signup.
- **401 Unauthorized:** `{"detail": "Invalid credentials"}` (header `WWW-Authenticate: Bearer`). Returned for both unknown email and wrong password.

```bash
curl -X POST http://localhost:8000/auth/login \
  -H 'Content-Type: application/json' \
  -d '{"email":"dev@applypilot.local","password":"DevTest1234!"}'
```

### `POST /auth/refresh`
- **Auth:** no (the refresh token is the credential) · **Plan:** none
- **Body** (`RefreshRequest`): `{ "refresh_token": string }`
- **Response 200** (`TokenPair`): a fresh access+refresh pair.
- **401 Unauthorized:** `{"detail": "Invalid refresh token"}` if the token is expired, malformed, or is an access token (wrong `type`).
- **Notes:** Tokens are not rotated/revoked server-side (see hardening backlog).

### `GET /auth/me`
- **Auth:** **yes** · **Plan:** none
- **Response 200** (`UserOut`):

```json
{
  "id": "0c2f...",
  "email": "dev@applypilot.local",
  "name": "Dev",
  "avatar_url": null,
  "plan": "free",
  "created_at": "2026-06-22T10:00:00+00:00"
}
```
- **401:** missing/invalid/expired token, or non-`access` token type.

## Resumes  (`/resumes`)

### `POST /resumes`
- **Auth:** **yes** · **Plan:** none
- **Body:** `multipart/form-data` with field `file`. Allowed content types: `application/pdf`, `application/msword`, `application/vnd.openxmlformats-officedocument.wordprocessingml.document` (PDF / DOC / DOCX).
- **Response 201** (`ResumeOut`):

```json
{ "id": "…", "filename": "resume.pdf", "storage_url": "http://minio:9000/applypilot/<user_id>/<uuid>-resume.pdf", "created_at": "…" }
```
- **422:** content type not in the allowed set (`{"detail": "Only PDF and Word documents are accepted"}`) or no file.
- **401:** unauthenticated.
- **Notes:** File is stored in MinIO under `<user_id>/<uuid>-<safe_filename>`; only metadata is in the DB. `parsed_text` and `embedding` are NULL in Phase 1.

```bash
curl -X POST http://localhost:8000/resumes \
  -H "Authorization: Bearer $TOKEN" \
  -F 'file=@resume.pdf;type=application/pdf'
```

### `GET /resumes`
- **Auth:** **yes** · **Plan:** none
- **Response 200:** `ResumeOut[]` — the caller's resumes, newest first.

### `DELETE /resumes/{resume_id}`
- **Auth:** **yes** · **Plan:** none
- **Response 204:** no content. Removes the MinIO object (idempotent) then the DB row.
- **404:** not found or owned by another user (`{"detail": "Resume not found"}`).

## Jobs  (`/jobs`)

Jobs are **global**, not user-scoped. In Phase 1 they are created manually / by the
seed script; Phase 3's scraper populates them. There is no update or delete endpoint.

### `POST /jobs`
- **Auth:** **yes** · **Plan:** none
- **Body** (`JobCreate`):

| Field | Type | Required | Notes |
|-------|------|----------|-------|
| `source` | string | yes | e.g. `greenhouse`, `lever`, `manual` |
| `company` | string | yes | |
| `role` | string | yes | |
| `jd_url` | string \| null | no | |
| `jd_text` | string \| null | no | Full JD (used for embedding in Phase 2/3) |
| `location` | string \| null | no | |
| `salary_range` | string \| null | no | |
| `posted_at` | datetime \| null | no | Part of the dedup key |

- **Response 201** (`JobOut`):

```json
{ "id":"…","source":"greenhouse","company":"Stripe","role":"Software Engineer",
  "jd_url":"https://…","location":"Remote","salary_range":null,
  "match_score":null,"posted_at":null,"status":"active" }
```
- **401:** unauthenticated.
- **Notes:** A unique constraint on `(company, role, posted_at)` exists; inserting a duplicate (same three values, non-NULL `posted_at`) raises a DB integrity error (HTTP 500 — there is no friendly 409 yet). `match_score` is NULL until Phase 2.

### `GET /jobs`
- **Auth:** **yes** · **Plan:** none
- **Query params:**

| Param | Type | Default | Notes |
|-------|------|---------|-------|
| `company` | string | — | Case-insensitive substring match |
| `source` | string | — | Exact match |
| `q` | string | — | Substring match on `role` OR `company` |
| `page` | int ≥ 1 | 1 | |
| `page_size` | int 1–100 | 20 | |

- **Response 200** (`JobList`): `{ "items": JobOut[], "total": int, "page": int, "page_size": int }`. Ordered by `scraped_at` descending.

```bash
curl -H "Authorization: Bearer $TOKEN" \
  "http://localhost:8000/jobs?company=stripe&page=1&page_size=20"
```

### `GET /jobs/{job_id}`
- **Auth:** **yes** · **Plan:** none
- **Response 200** (`JobOut`).
- **404:** `{"detail": "Job not found"}`.

## Applications  (`/applications`)

Full CRUD, scoped to the authenticated user. Cross-user access returns 404.

### `POST /applications`
- **Auth:** **yes** · **Plan:** none
- **Body** (`ApplicationCreate`): `{ "job_id": "<uuid>" }`
- **Response 201** (`ApplicationOut`) — status `pending`, with the embedded `job`:

```json
{
  "id":"…","job_id":"…","status":"pending",
  "email_subject":null,"email_body":null,"cover_letter":null,"linkedin_msg":null,
  "recruiter_email":null,"recruiter_linkedin":null,
  "sent_at":null,"follow_up_at":null,"reply_at":null,"created_at":"…",
  "job": { "id":"…","source":"…","company":"…","role":"…", "...": "…" }
}
```
- **404:** referenced job does not exist (`{"detail": "Job not found"}`).

### `GET /applications`
- **Auth:** **yes** · **Plan:** none
- **Query:** `status` (optional) — one of `pending|generated|sent|opened|replied|rejected|offer`. Invalid value → 422.
- **Response 200:** `ApplicationOut[]`, newest first, the caller's only.

### `GET /applications/{application_id}`
- **Auth:** **yes** · **Plan:** none
- **Response 200** (`ApplicationOut`).
- **404:** not found or not owned.

### `PATCH /applications/{application_id}`
- **Auth:** **yes** · **Plan:** none
- **Body** (`ApplicationUpdate`, all optional — only present fields are applied):

| Field | Type | Notes |
|-------|------|-------|
| `status` | enum | `pending\|generated\|sent\|opened\|replied\|rejected\|offer`. **No transition rules** — any valid value is accepted. |
| `email_subject` | string \| null | |
| `email_body` | string \| null | |
| `cover_letter` | string \| null | |
| `linkedin_msg` | string \| null | |
| `recruiter_email` | string \| null | |
| `recruiter_linkedin` | string \| null | |

- **Response 200** (`ApplicationOut`) — the updated record.
- **404:** not found or not owned. **422:** invalid `status` enum value.

```bash
curl -X PATCH http://localhost:8000/applications/$ID \
  -H "Authorization: Bearer $TOKEN" -H 'Content-Type: application/json' \
  -d '{"status":"sent"}'
```

### `DELETE /applications/{application_id}`
- **Auth:** **yes** · **Plan:** none
- **Response 204:** no content. **404:** not found or not owned.

## Dashboard  (`/dashboard`)

### `GET /dashboard/stats`
- **Auth:** **yes** · **Plan:** none
- **Response 200** (`DashboardStats`):

```json
{
  "total_applications": 3,
  "by_status": { "pending": 1, "generated": 1, "sent": 1 },
  "reply_rate": 0.0,
  "recent": [ /* up to 5 ApplicationOut, newest first */ ]
}
```
- **Notes:** All counts scoped to the caller. `reply_rate` = (applications with status `replied` or `offer`) / total, rounded to 4 dp; `0.0` when the user has no applications.
